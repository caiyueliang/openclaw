## 关键经验总结

### 1. Docker 启动命令（带自动重启和目录挂载）

```powershell
docker run -itd --ipc=host `
  --name openclaw-gateway `
  --restart always `
  -p 18789:18789 `
  -p 18790:18790 `
  -v ~:/root `
  tck-xinan-registry.cn-chengdu.cr.aliyuncs.com/wair/openclaw:1.0.5
```

**参数说明：**
- `--restart always` - 容器自动重启
- `-v ~:/root` - 挂载本地用户目录到容器的 /root
- `-p 18789:18789` - 映射网关端口
- `--ipc=host` - 共享主机 IPC 命名空间

### 2. 容器内启动 Gateway 服务

进入容器后执行：

```bash
# 启动 gateway（绑定到所有接口，使用 token 认证）
nohup pnpm openclaw gateway --port 18789 --bind lan --token openclaw123 --verbose > /tmp/gateway.log 2>&1 &

# 或者使用配置文件方式（推荐）
pnpm openclaw config set gateway.mode local
pnpm openclaw config set gateway.bind lan
pnpm openclaw config set gateway.port 18789
pnpm openclaw config set gateway.auth.mode token
pnpm openclaw config set gateway.auth.token openclaw123
pnpm openclaw config set gateway.controlUi.allowInsecureAuth true

# 然后启动
nohup pnpm openclaw gateway --verbose > /tmp/gateway.log 2>&1 &
```

### 3. 关键配置参数说明

| 参数 | 说明 | 值 |
|------|------|-----|
| `gateway.bind` | 绑定模式 | `lan` (0.0.0.0) / `loopback` (127.0.0.1) |
| `gateway.auth.mode` | 认证模式 | `token` / `password` |
| `gateway.auth.token` | 认证令牌 | 自定义字符串 |
| `gateway.controlUi.allowInsecureAuth` | 允许不安全认证（跳过设备配对） | `true` / `false` |

**注意：** `allowInsecureAuth: true` 用于开发环境，允许通过 token 直接连接而无需设备配对。

### 4. 常见问题及解决

#### 问题1：端口访问不了（Connection refused）

**原因**：服务绑定到 `127.0.0.1`，外部无法访问

**解决**：使用 `--bind lan` 绑定到 `0.0.0.0`

#### 问题2：unauthorized: gateway token missing (1008)

**原因**：客户端未提供 token

**解决**：
- URL 方式：`http://localhost:18789/?token=openclaw123`
- 或在 Trae 设置中填入 Token

#### 问题3：disconnected (1006): no reason

**原因**：WebSocket 连接异常断开，可能是服务崩溃或配置错误

**解决**：
```bash
# 查看服务状态
docker exec openclaw-gateway ps aux | grep openclaw

# 查看日志
docker exec openclaw-gateway tail -20 /tmp/openclaw/openclaw-2026-02-08.log
```

#### 问题4：pairing required (1008)

**原因**：OpenClaw 需要设备配对授权

**解决**：
```bash
# 禁用设备配对检查（开发环境）
pnpm openclaw config set gateway.controlUi.allowInsecureAuth true

# 然后重启 gateway
pkill -f 'openclaw-gateway'
nohup pnpm openclaw gateway --verbose > /tmp/gateway.log 2>&1 &
```

### 5. 常用命令

```bash
# 查看容器状态
docker ps | findstr openclaw-gateway

# 进入容器
docker exec -it openclaw-gateway bash

# 查看 gateway 日志
docker exec openclaw-gateway tail -f /tmp/openclaw/openclaw-2026-02-08.log

# 查看进程
docker exec openclaw-gateway ps aux | grep openclaw

# 停止 gateway 服务
docker exec openclaw-gateway pkill -f 'openclaw-gateway'

# 重启容器
docker restart openclaw-gateway

# 删除容器
docker rm -f openclaw-gateway
```

### 6. 配置文件位置

容器内配置文件路径：
```
/root/.openclaw/openclaw.json
```

关键配置项：
```json
{
  "gateway": {
    "mode": "local",
    "bind": "lan",
    "port": 18789,
    "auth": {
      "mode": "token",
      "token": "openclaw123"
    },
    "controlUi": {
      "enabled": true,
      "allowInsecureAuth": true
    }
  }
}
```

### 7. Trae 连接配置

在 Trae 中配置 OpenClaw：

1. **方式一**：直接访问带 token 的 URL
   ```
   http://localhost:18789/?token=openclaw123
   ```

2. **方式二**：在设置中配置
   - Gateway URL: `http://localhost:18789`
   - Token: `openclaw123`
